01 · Abstrak

Latar Belakang & Rumusan Masalah

Hard Disk Drive (HDD) tetap menjadi tulang punggung penyimpanan data di pusat data dan sistem enterprise. Seiring bertambahnya usia dan densitas, HDD rentan mengalami bad sector — area yang gagal dibaca secara andal. Membaca sebuah bad sector membutuhkan waktu hingga 8 detik per sektor akibat mekanisme read retry dan ECC.

Permasalahan inti: dalam urutan apa sektor-sektor target harus dikunjungi agar total waktu pemulihan seminimal mungkin? Ini adalah varian Travelling Salesman Problem (TSP) pada ruang linear satu dimensi (LBA), yang termasuk kelas kompleksitas NP-Hard.

Hybrid DP (Held-Karp)

Solusi yang Diusulkan

Framework dua-tahap: Dijkstra membangun matriks jarak antar sektor penting, lalu DP bitmask menentukan urutan kunjungan optimal. Menjamin solusi exact optimal.

Greedy Nearest Neighbor

Baseline Pembanding

Selalu mengunjungi sektor terdekat berikutnya dari posisi saat ini. Kompleksitas O(k²) — cepat, namun tidak menjamin optimalitas, terutama saat distribusi target asimetris.

Model LOGI

Simulasi Bad Sector

Menggunakan model LOGI dari Ahmad et al. — BFS berbobot yang menyebarkan kerusakan ke sektor tetangga (LBA±1) dengan probabilitas cluster_prob, merepresentasikan spatial locality.

Formula Bobot Edge

w(i → j) = SECTOR_SIZE_MB / min(read_speed, throughput) + penalty(j)
penalty(j) = retry_time = 8.0 detik jika j adalah bad sector, else 0
SECTOR_SIZE_MB = 512 bytes = 4.768e-4 MB

Seluruh bobot bersifat non-negatif → Dijkstra dapat diterapkan dengan jaminan optimalitas. Bad sector di antara dua titik menyebabkan jarak temporal yang jauh lebih besar dari jarak spasial — inilah alasan Dijkstra dibutuhkan, bukan sekadar kalkulasi linear.

Pipeline Eksekusi

1a

Bangkitkan Graf HDD dengan Model LOGI

Tanamkan seed bad sector secara acak, sebarkan ke tetangga (LBA±1) dengan probabilitas cluster_prob menggunakan BFS hingga target bad_ratio tercapai. Kompleksitas: O(N).

HDDState ← generate_logi(HDDConfig)
1b

Bangun Adjacency List & Jalankan Dijkstra

Graf linear bidirectional: setiap LBA terhubung ke LBA±1 dengan bobot berdasarkan formula di atas. Dijkstra dijalankan dari setiap node penting (start + k target) — total (k+1) eksekusi.

dist_matrix[(k+1)×(k+1)] ← build_distance_matrix(adj, [s, t₁..tₖ])
2

DP Bitmask (Held-Karp) — Urutan Optimal

dp[mask][last] = waktu minimum mengunjungi tepat set sektor yang direpresentasikan mask, dengan last sebagai sektor terakhir. Transisi: tambahkan target berikutnya yang belum dikunjungi.

dp[mask | (1<<j)][j] = min(dp[mask][last] + dist[last+1][j+1])

Pseudocode — Held-Karp DP

def dp_tsp(dist_matrix, k):
    FULL = (1 << k) - 1
    dp     = [[∞] * k for _ in range(1 << k)]
    parent = [[-1] * k for _ in range(1 << k)]

    # Inisialisasi: dari start (node 0) ke setiap target
    for i in range(k):
        dp[1 << i][i] = dist_matrix[0][i + 1]

    # Isi tabel DP — semua subset mask
    for mask in range(1, 1 << k):
        for last in range(k):
            if dp[mask][last] == ∞: continue
            for nxt in range(k):
                if (mask >> nxt) & 1: continue
                new_mask = mask | (1 << nxt)
                new_time = dp[mask][last] + dist_matrix[last+1][nxt+1]
                if new_time < dp[new_mask][nxt]:
                    dp[new_mask][nxt]     = new_time
                    parent[new_mask][nxt] = last

    # Rekonstruksi urutan optimal
    best = min(range(k), key=lambda l: dp[FULL][l])
    return dp[FULL][best], reconstruct(parent, FULL, best)

06 · Analisis Kompleksitas

Analisis Waktu & Ruang

Masalah ini termasuk kelas NP-Hard melalui reduksi polinomial dari TSP. Namun, dengan DP Held-Karp dan batasan k yang kecil, solusi optimal tetap komputasional.

Bottleneck DP

Untuk k=15: 2^15 = 32.768 state × 15² = 7.372.800 operasi DP. Pada hardware modern, ini selesai dalam <500ms — jauh di bawah total recovery time yang bisa mencapai ribuan detik.

k=15 → ~7.3M operasi → <500ms

Kelas Kompleksitas

Masalah termasuk NP-Hard karena dapat direduksi dari TSP secara polinomial. Namun, karena k biasanya kecil (3–15 file target), DP exact tetap praktis digunakan.

TSP ≤p HDD-Recovery-Ordering ∈ NP-Hard

Trade-off Praktis

Greedy relevan saat k > 15 atau latensi komputasi kritis. Hybrid DP unggul saat k ≤ 15 dan optimalitas penting (bad_ratio tinggi, file kritikal).

07 · Kesimpulan

Temuan & Kontribusi Penelitian

✅

DP selalu optimal

Held-Karp menjamin solusi exact. Improvement ≥ 0% di semua skenario — tidak pernah lebih buruk dari Greedy.

📈

Signifikan saat bad_ratio tinggi

Improvement DP paling besar pada bad_ratio tinggi (12.07%) dan k besar — distribusi penalti membuat urutan kunjungan sangat kritis.

⏱️

Komputasi tetap praktis

CPU time DP < 500ms untuk k ≤ 15, jauh di bawah total recovery time yang mencapai ribuan detik.

🔀

Greedy tetap relevan

Untuk k > 15 atau skenario real-time, Greedy adalah pilihan pragmatis meski tidak menjamin optimalitas.

Kontribusi Utama Penelitian

1

Model Komputasional Baru

Formulasi masalah scanning path HDD recovery yang secara eksplisit mempertimbangkan offset sektor, propagasi bad sector (LOGI), kecepatan baca, dan throughput scanner — lebih realistis dari pendekatan binary.

2

Framework Hybrid Pertama di Domain HDD Recovery

Pertama kali mengintegrasikan Dijkstra (shortest path lokal) + DP bitmask (urutan global optimal) pada domain data recovery HDD — mengisi gap yang belum pernah diisi dalam literatur.

3

Implementasi & Analisis Lengkap

Python reproducible dengan seed deterministik, analisis kompleksitas komprehensif, reduksi ke NP-Hard, dan rekomendasi praktis bagi pengembang data recovery dan praktisi digital forensics.
